JavaScript基础

  1. 五种变量类型：number, string, boolean, NaN, undefined

  2. 一元运算符中有两个特别的运算符，一个是正号：+，一个是负号：-

  3. 在JS中，如果运算数不是运算符所要求的类型，那么JS引擎会自动地将运算数进行类型转换，转换规则如下：

    • string 转 number，按照字面值转换，例如”123“转换为数字类型的123；如果字面值不是数字，则转换为NaN(Not a Number表示不是数字的数字)，例如”abc“转换为NaN
    • boolean 转 number，规则为true转换为1，false转换为0

  4. == 和===

    • == 如果类型相同则直接比较，如果类型不同，则先进行类型转换，比如“123” == 123 则返回true
    • === 全等于，在比较之前先判断类型，类型不同直接返回false，比如“123” ===123 则 返回false

  5. var 关键字变量声明

事件

  1. 一些概念：

    • 事件：指某些操作。如：单击，双击，键盘按下，鼠标移动
    • 事件源：指事件发生的位置。如：按钮，文本输入框
    • 监听器：指事件发生后执行的代码
    • 注册监听：将事件，事件源，监听器结合在一起，当事件源上发生了某个事件，则触发某个监听器代码

  2. 一些事件

    • 点击事件：onclick单击事件，ondbclick双击事件
    • 焦点事件：onblur失去焦点，onfocus获得焦点。比如光标在一个地方闪就是获得焦点
    • 加载事件：onload页面或图片完成加载
    • 鼠标事件：onmousedown鼠标按钮被按下，onmouseup鼠标按钮被松开，onmousemove鼠标被移动，onmouseout鼠标从某元素移开，onmouseover鼠标移到某元素上
    • 键盘事件：onkeydown某个键盘按键被按下，onkeyup某个键盘按键被松开，onkeypress某个键盘按键被按下并松开
    • 选中和改变：onchange域的内容被改变，onselect文本 被选中
    • 表单事件：onsubmit确认按钮被点击，onreset重置按钮被点击

  3. onmousedown事件，在定义方法时定义一个形参，接收event对象，event对象的button属性可以获取哪个鼠标按钮键被点击了

Bootstrap

  1. 响应式布局：即同一套页面可以兼容不同的分辨率（即不同大小）的设备。主要依赖于栅格系统：栅格系统将一行平均分为12个格子，可以指定元素占几个格子

    • 定义容器。(相当于之前的table)容器有两种，一种是container固定宽度，两边留白，一种是container-fluid自动占满浏览器屏幕宽度

    • 定义行。(相当于之前的tr)采用样式：row

    • 定义元素。指定该元素在不同的设备上，所占的格子数目。采用样式：col-设备代号-格子数目。

      设备代号：xs手机，sm平板，md笔记本电脑桌面，lg台式机电脑桌面

xml（Extensible Markup Language）

  1. 概念：xml可扩展标记语言是用来存储数据的，可以作为一种配置文件，在网络中传输，并且有严格的语法格式，xml和html有相似之处，都可以被浏览器解析，但是不像html的p标签，div标签等，xml的标签是可以自定义的。

  2. 基本语法

    • xml文档后缀名为.xml
    • xml第一行必须为文档声明
    • xml文档中有且仅有一个根标签
    • 属性值必须使用引号引起来
    • 标签必须正确关闭
    • xml标签名区分大小写

  3. 组成部分

    • 文档声明

      属性列表为：version版本号，encoding编码方式，standalone是否独立

    • 指令

      和html一样，结合css可以控制标签样式

    • 标签

    • 属性

      id属性值必须唯一

    • 文本

      CDATA区：在该区域中的数据会原样展示

  4. 约束：约束规定了xml文档的书写规则，要求会看懂约束，并会根据约束写xml文档 即可

    • DTD:一种简单的约束技术，后缀.dtd

    • Schema:一种复杂的约束技术，后缀.xsd

  5. 解析：操作xml文档，将文档中的数据读取到内存中

    解析xml的方式：

    • DOM：将标记语言文档一次性加载进内存，在内存中形成一个树形结构DOM树

      其优点是操作方便，可以对文档进行CRUD的所有操作

      其缺点是占内存，一个小的标记语言文档形成一颗DOM树时，其大小会扩大1000倍左右

    • SAX：逐行读取，基于事件驱动

      其优点是不占内存

      其缺点是只能读取，不能增删改

    常见的解析器：

    • JAXP：支持dom和sax两种思想，很烂
    • DOM4J：一款优秀的解析器
    • Jsoup：本来是用来解析HTML的，但是它也可以用来解析xml
    • PULL：安卓操作系统内置的解析器，sax方式

Servlet(server applet)
  1. 概念：运行在服务器端的小程序
    • servlet就是一个接口，定义了Java类被浏览器访问到(即被tomcat识别)的规则
    • 将来我们自定义一个类，实现servlet接口，覆写方法
  2. servlet的体系结构
    • GenericServlet类，将Servlet接口中其他方法做了默认空实现，只需要实现service方法即可
    • HttpServlet类是对http协议的一种封装，简化操作，需要实现doGet和doPost方法
HTTP

  1. HTTP是基于TCP/IP的高级协议，其默认端口号是80，是基于请求/响应模型，就是说一次请求对应一次响应，并且每次请求之间相互独立，不能交换数据

  2. 请求消息数据格式

    • 请求行

      请求方式 请求url 请求协议/版本。

      http有7种请求方式，常用的有2种get和post

    • 请求头

      请求头名称：请求头值

      User-Agent：可以在服务器端获取该头的信息，解决浏览器的兼容性问题

      Referer：主要是防盗链和计数工作

    • 请求空行

      用于分隔POST请求的请求头和请求体

    • 请求体

      get方式没有请求体，post有，比如username = zhangsan

      用于封装POST请求消息的请求参数

  3. 响应消息数据格式

    • 响应行

      协议/版本 响应状态码 状态码描述

      响应状态码大体上有五种分类：1xx，2xx，3xx，4xx，5xx

      1xx：服务器接收客户端消息，但是没有接收完成，等待一段时间后发送该类型状态码

      2xx：成功。典型的例子是200

      3xx：重定向。典型的例子是302（重定向），304（访问缓存）

      4xx：客户端错误。典型的例子是404(请求路径没有对应资源)，405(请求方式没有对应的doxxx方法)

      5xx：服务端错误。典型的例子是500(服务器内部出现异常)

    • 响应头

      头名称：值

      Content-Type：服务器告诉客户端本次响应消息体数据格式和编码格式

      Content-disposition：服务器告诉客户端以什么格式打开响应体数据

    • 响应空行

    • 响应体

      传输的数据

  4. get和post区别

    • get：
      1. 请求参数在地址栏中显示，会封装在请求行中
      2. 请求参数大小有限制
      3. 不太安全
    • post：
      1. 请求参数不会再地址栏中显示，会封装在请求体中
      2. 请求参数的大小没有限制
      3. 较为安全
Request

  1. 获取请求行数据，比如请求行: GET /day13/demo1?name=zhangsan HTTP/1.1

    • 获取请求方式的方法：String getMethod()。返回GET

    • (重点)获取虚拟目录的方法：String getContextPath()。返回/day13

    • 获取Servlet的路径：String getServletPath()。返回/demo1

    • 获取get方式的请求参数：String getQueryString()。返回name = zhangsan

    • (重点)获取请求URI：String getRequestURI()。返回/day13/demo1

    • 获取请求URL：StringBuffer getRequestURL()。返回http://localhost/day13/demo1

    • 获取协议和版本：String getProtocol()。返回HTTP/1.1

    • 获取客户机的IP地址：String getRemoteAddr()

  2. 获取请求头数据，

    • (重点)通过请求头的名称获取请求头的值：String getHeader(String name)
    • 获取所有的请求头名称：Enumeration<String> getHeaderNames

  3. 获取请求体数据，只有POST才有请求体，在请求体中封装了POST的请求参数

    • 获取字符输入流，只能操作字符数据：BufferedReader getReader()
    • 获取字节输入流，可以操作所有类型数据：ServletInputStream getInputStream()

  4. 获取请求参数的通用方式：无论是get还是post

    • (重点)根据参数名称获取参数值：String getParameter(String name)

    • 根据参数名称获取参数值数组，多用于复选框：String[] getParameterValues(String name)

    • 获取所有的请求参数名称：Enumeration<String> getParameterNames()

    • (重点)获取所有参数的键值对集合：Map<String, String[]> getParameterMap()

  5. 请求转发，一种在服务器内部的资源跳转方式

    • 通过request对象获取请求转发器对象：RequestDispatcher getRequestDispatcher(String path)
    • 使用RequestDispatcher对象来进行转发：forward(ServletRequest request, ServletResponse response)
    • 请求转发的特点：1. 浏览器的地址栏不发生改变；2. 只能将请求转发到服务器内部的资源中；3. 转发是一次请求

  6. 共享数据，是一个有作用范围的对象，可以在范围内共享数据。而request域代表一次请求的范围，一般用于共享数据

    • 存储数据到request域中：void setAttribute(String name, Object obj)
    • 通过键获取值：Object getAttribute(String name)
    • 通过键移除键值对：void removeAttribute(String name)

  7. 获取ServletContext

    • ServletContext getServletContext()

Cookie

  1. 概念：是一种客户端会话技术，将数据保存到客户端

  2. 使用步骤：

    • 创建Cookie对象，绑定数据

      new Cookie(String name, String value)

    • 发送Cookie对象

      response.addCookie(Cookie cookie)

    • 获取Cookie，拿到数据

      Cookie[] request.getCookies()

  3. 一次可以发送多个Cookie，可以创建多个Cookie对象，使用response对象多次使用addCookies方法发送Cookie

  4. 默认情况下，浏览器关闭后，Cookie数据被销毁，如想要持久化存储，则需要使用如下方法

    setMaxAge(int seconds)

    如果传入的参数是正数，则表示将Cookies数据写到硬盘的文件中，该正数表示存活时间，也就是持久化时间

    如果传入的参数是负数，是默认值

    如果传入的参数是0，则表示删除Cookie信息

  5. 在tomcat 8 之前 cookie不支持中文，需要将中文数据转码，一般采用URL编码。在tomcat 8 之后，cookie支持中文，不支持特殊字符，建议使用URL编码存储，并用URL解析

  6. 在一个tomcat服务器中，部署了多个web项目，那么在web项目间的cookie默认不能共享，但是可以采用如下方式扩大共享范围

    setPath(String path)：设置cookie的获取范围，默认情况下，path是当前的虚拟目录

    如果要不同项目间可以共享，则可以采用setPath(“/”)，意思是服务器根目录下的所有项目都可以共享

  7. 而要想在不同的tomcat服务器之间共享cookie，则采用如下方法

    setDomain(String path)：如果一级域名相同则多个服务器之间cookie可以共享，比如setDomain(".baidu.com")则tieba.baidu.com和news.baidu.com两个服务器之间cookie可以共享

  8. 浏览器对单个cookie大小有限制（4kb）以及对同一个域名下的总cookies数量也有限制（20个）

JSP(已过时)

  1. JSP(Java Server Pages)是一个特殊的页面，其中既可以定义Html标签，也可以写java代码

  2. Jsp中定义java代码的方式

    • <% 代码 %>：定义的java代码，在service方法中。service方法中可以定义什么，该脚本中就可以定义什么
    • <%! 代码 %>：定义的java代码，在jsp转换后的java类的成员位置
    • <%= 代码 %>：定义的java代码，会输出到页面上，输出语句中可以定义什么，该脚本中就可以定义什么

  3. Jsp的内置对象，在jsp页面不需要获取和创建，可以直接使用的对象

Session

  1. 概念：是一种服务器端的会话技术，在一次会话的多次请求间共享数据，将数据保存在服务器端对象中

  2. 使用步骤：

    • 获取HttpSession对象

      HttpSession session = request.getSession();

    • 使用HttpSession对象

      Object getAttribute(String name)

      void setAttribute(String name, Object value)

      void removeAttribute(String name)

    • 当客户端关闭后，服务器不关闭，两次获取的session默认情况下不是同一个。可以创建一个cookie，键为JSEESIONID，设置最大存活时间，让cookie持久化存储

      Cookie c = new Cookie("JSESSIONID"，session.getId());

    • 客户端不关闭，服务器关闭后，两次获取的session不是同一个，但是需要确保数据不丢失，tomcat服务器自动帮我们做了如下两件事

      session钝化：在服务器正常关闭之前，将session对象系列化到硬盘上

      session活化：在服务器启动后，将session文件转化为内存中的session对象

    • session在以下情况下时自动销毁

      服务器关闭

      session对象调用invalidate()

      session默认失效时间：30分钟

    • session用于存储一次会话的多次请求的数据，存在服务器

    • session可以存储任意类型，任意大小的数据

Filter
  1. filter的作用是完成通用的操作，比如，登录验证，统一编码处理，敏感字符过滤等.......
  2. 使用步骤：1）定义一个类，实现接口Filter；2）覆写方法；3）配置拦截路径
  3. 拦截路径有几种方式
    • 具体资源路径：/index.jsp 只有访问index.jsp资源时，过滤器才会被执行
    • 拦截目录：/user/* 访问/user下的所有资源时，过滤器都会被执行
    • 后缀名拦截：*.jsp 访问所有后缀名为jsp的资源时，过滤器都会被执行
    • 拦截所有资源：/* 访问所有资源时，过滤器都会被执行
  4. 资源访问方式的配置，可以通过在注解中设置dispatcherTypes属性
    • REQUEST：默认值，浏览器直接请求资源，比如浏览器直接请求index.jsp时过滤器才会被执行
    • FORWARD：转发访问资源，比如只有转发请求index.jsp时过滤器才会被执行
    • INCLUDE：包含访问资源，
    • ERROR：错误跳转资源
    • ASYNC：异步访问资源
  5. 过滤器先后顺序如果按照注解配置，那么是按照类名的字符串比较规则比较 ，值小的先执行

    • 配置（注解配置，webxml配置）

    • 小于索引选择器

    • CRUD操作

      appendTo(): 比如 对象1.appendTo(对象2) 将对象1添加到对象2内部，并且在末尾

      prependTo()：比如 对象1.prependTo(对象2) 将对象1添加到对象2内部，并且在开头

      insertAfter(): 比如 对象1.insertAfter(对象2) 将对象1添加到对象2后边。对象1和对象2是兄弟关系

redis

  1. redis是属于NoSQL系列的非关系型数据库，它存储的是键值对，数据之间没有关联关系，最重要的一点是，非关系型数据库存储在内存中，而关系型数据库存储在硬盘文件中。

  2. redis存储的是key:value格式的数据，其中key都是字符串类型，而value有5种不同类型的数据结构

    • 字符串类型 string
    • 哈希类型 hash（map的格式）
    • 列表类型 list （linkedlist格式）
    • 集合类型 set
    • 有序集合类型 sortedset

  3. 基本的命令（参考redis中文网redis教程）

    字符串类型 string

    • 存储 set age 13 返回OK
    • 获取 get age 返回 “23”
    • 删除 del age

    哈希类型 hash

    • 存储 hset key field value 比如 hset myhash username lisi
    • 获取 hget key field 比如 hget myhash username 返回 “lisi” （hgetall key 获取所有的field 和value）
    • 删除 hdel key field

    列表类型 list

    • 添加 lpush key value: 将元素加入列表左边；rpush key value: 将元素加入列表右边
    • 获取 lrange key start end: 范围获取
    • 删除 lpop key: 删除列表最左边的元素并返回；rpop key: 删除列表最右边的元素并返回

    集合类型 set

    • 存储 sadd key value 比如 sadd myset a b c: 可以一次性加入三个元素
    • 获取 smembers key: 获取set集合中所有元素
    • 删除 srem key value: 删除set集合中的某个元素

    有序集合 sortedset

    • 存储 zadd key score value: 存储数据，并存储数据对应的分数，根据分数排序
    • 获取 zrange key start end
    • 删除 zrem key value

  4. 通用命令

    • keys *: 获取所有的键
    • type key: 根据键名获取其值的类型
    • del key: 删除指定的key: value

  5. redis持久化

    • RDB：默认方式，不需要进行配置，在一定的间隔时间中，检测key的变化情况，然后持久化数据

    • AOF：日志记录的方式，可以记录每一条命令的操作，每一次命令操作后持久化数据
